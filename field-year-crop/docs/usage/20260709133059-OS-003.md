# OS-003 Usage Note

Run the local synthetic workflow smoke from the LandCore repo root:

```bash
bash field-year-crop/scripts/smoke/local_synthetic_field_crop_year.sh
```

The smoke runs under WSL, builds `go-etl` binaries from the sibling `../go-etl` checkout,
starts the controller, submits the LandCore project and workflow files through
`goet submit --project --workflow`, and lets the controller start workers through its
direct-process scheduler. The workflow `source_manifest` is what causes the client to
include the Python source files needed by the workers.

Expected outputs:

- `field_crop_year_counts.csv`
- `field_crop_year_summary.csv`

Useful log locations during a failed run:

- client: `demo.log` in the smoke temp directory
- controller: `controller.log` in the smoke temp directory
- worker log: `controller-runtime/logs/worker.log`
- worker attempt logs: `controller-runtime/tmp/attempts/<attempt-id>/logs/`

The smoke uses tiny synthetic rasters in `/tmp/landcore-field-year-crop/local-synthetic` and
does not require real CDL or Yan/Roy inputs.
